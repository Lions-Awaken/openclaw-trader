What is openclaw-trader?

openclaw-trader is an autonomous AI swing trading system running 24/7 on ridley — a NVIDIA Jetson Orin Nano 8GB. It monitors a 39-ticker watchlist of AI infrastructure stocks (NVDA, AMD, AVGO, SMCI, MRVL and others), identifies high-probability swing setups using a multi-stage AI reasoning chain, and executes paper trades via the Alpaca API.

The system is built around an adversarial ensemble: one live profile makes trading decisions, while 6 shadow agents run in parallel, silently evaluating the same candidates and recording their disagreements. Those disagreements become training data for a weekly calibration loop that learns which agents are consistently right.

The system is conservative by design. When in doubt, it does nothing. Cash is a position.

The Scanner Pipeline — T1 through T5 Tumbler Chain

T1 Signal Scoring FREE

39 watchlist tickers scored against 6 binary signals: Trend, Momentum, Volume, Catalyst, Sentiment, Flow

  • Score 0–6. Only tickers scoring ≥ 3 advance.
  • Pure computation — no AI inference involved.
T2 Fundamental Analysis (RAG) FREE
  • Retrieves similar past inference chains and trade learnings via pgvector similarity search
  • Congressional trade disclosures get confidence boost if filed within 40 days
  • Hard veto if sentiment score < −0.5
T3 Flow & Cross-Asset (Ollama on ridley GPU) FREE (local)
  • First LLM call — runs locally on ridley's Jetson GPU via Ollama (qwen2.5:3b)
  • Analyzes how this setup compares to past chains and outcomes
  • Confidence adjustment: ±0.10
T4 Pattern Match (Claude Haiku — Anthropic API) ~$0.001/call
  • Matches against known pattern templates with documented outcomes
  • Evaluates quality and coherence of the full trade thesis
  • Confidence adjustment: ±0.10
T5 Counterfactual (Claude Sonnet — Anthropic API) ~$0.005/call
  • Devil's advocate — constructs the strongest argument AGAINST the trade
  • Asymmetric adjustment: can drop confidence by 0.15 but raise it only 0.05
  • Most expensive tumbler; only fires when budget allows
GATE Execution Gate

Trade only executes if ALL THREE conditions are met:

  • Decision = enter or strong_enter
  • Final confidence ≥ 0.60
  • At least 3 tumblers completed

Stopping rules: veto_signal   confidence_floor   time_limit   resource_limit   congress_signal_stale

The Calibration Loop — DWM

Every Sunday, the calibrator grades all ungraded divergences from the past 30 days:

  1. Fetches ungraded shadow_divergences where shadow_was_right is null
  2. Grades each: was the shadow agent right to dissent?
    • Kronos: checks actual close price at 10 trading days after signal
    • LLM agents: checks inference_chains.actual_outcome
  3. Fitness score: fitness = times_correct / times_dissented
  4. DWM weight: new_weight = 1.0 × (1 + 0.5 × (fitness − median_fitness))  clamped [0.05, 3.0]

Meta Daily Reflection — 1:30 PM

Every afternoon, Claude reviews the day's shadow divergences looking for unanimous dissent — all 6 shadows simultaneously disagreed with the live profile. If found:

1 Full divergence data injected into Claude Sonnet prompt as HIGH PRIORITY
2 Instruction: "Analyze what the shadows saw that the live profile missed"
3 Output written to meta_reflections table
4 The reflection becomes RAG context for future T2 tumbler calls

Circuit Breakers and Risk Limits

  • VIX > 28 → scanner stands down for the session
  • Portfolio drawdown > 15% → circuit breaker trips
  • 3 consecutive losses → circuit breaker trips
  • Max 3 concurrent positions
  • 5% max risk per trade (ATR-based sizing)
  • 10-day max hold period (time stop)
